2.2.3 Menginstal Django

Ada tiga cara untuk menginstal Django:

Menginstal rilis resmi.

2.3.1 Membuat Proyek

Jika ini pertama kalinya Anda menggunakan Django, ada beberapa langkah awal yang perlu dilakukan.
Anda perlu membuat kode otomatis yang membentuk sebuah proyek Django — yaitu kumpulan pengaturan untuk sebuah instance Django, termasuk konfigurasi database, opsi khusus Django, dan pengaturan untuk aplikasi.

Dari command line, masuk ke direktori tempat Anda ingin menyimpan kode, lalu buat direktori baru bernama djangotutorial.
(Nama direktori ini tidak berpengaruh pada Django; Anda bisa menamainya sesuka hati.)

$ mkdir djangotutorial


Kemudian jalankan perintah berikut untuk membuat proyek Django baru:

$ django-admin startproject mysite djangotutorial


Perintah ini akan membuat proyek bernama mysite di dalam folder djangotutorial.
Jika perintah ini tidak berhasil, lihat bagian Problems running django-admin.

?? Catatan:
Hindari memberi nama proyek dengan nama komponen bawaan Python atau Django.
Misalnya, jangan gunakan nama django (akan bertabrakan dengan Django itu sendiri) atau test (karena konflik dengan modul Python bawaan).

?? Struktur Folder yang Dihasilkan
djangotutorial/
    manage.py
    mysite/
        __init__.py
        settings.py
        urls.py
        asgi.py
        wsgi.py

Penjelasan:

manage.py
Utilitas baris perintah untuk berinteraksi dengan proyek Django.
Anda bisa membaca detail lengkapnya di dokumentasi django-admin and manage.py.

mysite/
Direktori ini adalah paket Python dari proyek Anda.
Nama folder inilah yang digunakan untuk mengimpor apa pun di dalamnya (misalnya mysite.urls).

mysite/init.py
File kosong yang memberi tahu Python bahwa folder ini adalah paket Python.
(Jika Anda baru mengenal Python, baca lebih lanjut tentang paket di dokumentasi resmi Python.)

mysite/settings.py
Berisi konfigurasi untuk proyek Django Anda (termasuk database dan opsi lainnya).

mysite/urls.py
Deklarasi URL untuk proyek Django Anda — semacam “daftar isi” untuk situs web Anda.

mysite/asgi.py
Titik masuk (entry point) untuk server web yang mendukung ASGI.

mysite/wsgi.py
Titik masuk untuk server web yang mendukung WSGI.

2.3.3 Membuat Aplikasi “Polls”

Sekarang lingkungan proyek sudah siap, saatnya mulai bekerja!

Setiap aplikasi di Django adalah paket Python yang mengikuti struktur tertentu.
Django memiliki utilitas yang secara otomatis membuat struktur dasar aplikasi agar Anda bisa langsung menulis kode tanpa repot menyiapkan folder secara manual.

?? Perbedaan antara Proyek dan Aplikasi

Aplikasi (app) adalah komponen yang melakukan sesuatu tertentu — misalnya sistem blog, database data publik, atau aplikasi polling sederhana.

Proyek (project) adalah kumpulan konfigurasi dan aplikasi untuk satu situs web tertentu.

Satu proyek dapat memiliki banyak aplikasi, dan satu aplikasi dapat digunakan di beberapa proyek.

Dalam tutorial ini, kita akan membuat aplikasi polls di dalam folder djangotutorial.

Untuk membuat aplikasi, pastikan Anda berada di direktori yang sama dengan manage.py, lalu jalankan:

$ python manage.py startapp polls


Ini akan membuat struktur direktori seperti berikut:

polls/
    __init__.py
    admin.py
    apps.py
    migrations/
        __init__.py
    models.py
    tests.py
    views.py


Struktur ini akan menjadi rumah bagi aplikasi polling Anda.

2.3.4 Menulis View Pertama Anda

Buka file polls/views.py, lalu tulis kode Python berikut:

from django.http import HttpResponse

def index(request):
    return HttpResponse("Hello, world. You're at the polls index.")


Ini adalah view paling dasar di Django.

Agar dapat diakses melalui browser, kita harus memetakan view ini ke sebuah URL.
Untuk itu, kita perlu membuat URL configuration (URLconf).

Buat file baru bernama polls/urls.py dan isi dengan kode berikut:

from django.urls import path
from . import views

urlpatterns = [
    path("", views.index, name="index"),
]


Sekarang, struktur folder polls/ Anda seharusnya terlihat seperti ini:

polls/
    __init__.py
    admin.py
    apps.py
    migrations/
        __init__.py
    models.py
    tests.py
    urls.py
    views.py


Langkah selanjutnya adalah menghubungkan URLconf aplikasi polls ke URL utama proyek mysite.
Buka file mysite/urls.py, lalu ubah menjadi seperti berikut:

from django.contrib import admin
from django.urls import include, path

urlpatterns = [
    path("polls/", include("polls.urls")),
    path("admin/", admin.site.urls),
]


Fungsi path() membutuhkan setidaknya dua argumen: route dan view.
Fungsi include() memungkinkan kita menyertakan URLconf lain.

Setiap kali Django menemukan include(), Django akan memotong bagian URL yang sudah cocok sejauh ini dan mengirimkan sisa string ke URLconf yang disertakan untuk diproses lebih lanjut.

Ide utama dari include() adalah memudahkan sistem URL agar dapat dipasang di berbagai lokasi — misalnya:

/polls/

/fun_polls/

/content/polls/

Semuanya dapat diarahkan ke aplikasi yang sama, cukup dengan mengubah jalur di urls.py.
